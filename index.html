FASE 1 - ANALISIS DE REQUERIMIENTOS PARA DESARROLLO WEB

Eddy Patricia Caceres Garcia

Terminos Web

Hosting:

Es un servicio de alojamiento web que permite publicar un sitio web o aplicación en Internet. Almacena todos los archivos y datos que componen un sitio web en un servidor para que el sitio web funcione correctamente.

Los proveedores de alojamiento web suelen ofrecer características como almacenamiento, cómputo y ancho de banda escalables, protocolos de seguridad, copias de seguridad, soporte técnico y otras opciones personalizables para satisfacer las necesidades del usuario. (IBM,2021)

Como obtener un hosting:

Para adquirir un hosting, se debe visitar el sitio web de un proveedor, seleccionar el plan de alojamiento web que se ajuste a las necesidades del cliente, añadirlo al carrito de compra y completar el proceso de pago con los datos personales. Luego, recibirá las credenciales para acceder al panel de control donde se podrá gestionar el sitio web, dominios, correos electrónicos y más.

Dominio:

Un nombre de dominio es la dirección física de tu sitio web. Ayuda a los usuarios a encontrar fácilmente el sitio en lugar de utilizar su dirección de protocolo de Internet (IP). Los nombres de dominio, formados por un nombre y una extensión, son una parte fundamental de la infraestructura de Internet. Por ejemplo, el nombre de dominio de Google es "google.com" .(Equipo editorial de IONOS,2024)

La traducción de los nombres de dominio en direcciones IP se realiza a través de los servidores DNS, servidores especializados responsables de la asignación de nombres a las direcciones IP. Este servicio funciona de manera similar a un servicio de información telefónica: un usuario escribe el dominio www.example.com en el campo de búsqueda de su navegador web y este envía la solicitud al servidor de nombres correspondiente.

Como se obtiene:

  • Elegir el dominio web: Como dirección web y herramienta de marketing muy importante, el nombre de dominio web debe ser único y acorde a la marca

    • Existen diferentes tipos de extensiones de dominio:

    • Dominio de nivel superior genérico (gTLD): el tipo de TLD más común para fines generales. Algunos ejemplos populares son .com, .org y .net. Extensiones más recientes como .xyz, .club y .online

    • Dominio de nivel superior de código de país (ccTLD): dominios de nivel superior de dos letras que representan a un país o ubicación geográfica concretos, como .us y .uk. Dado que las autoridades locales los regulan, el proceso de registro puede tener requisitos y restricciones específicos.
    • Dominio de nivel superior patrocinado (sTLD): diseñado para entidades específicas. Por ejemplo,.edu para organizaciones educativas, mientras que .gov está reservado para organismos gubernamentales.

  • Elegir un registrador de dominios fiable: Un registrador de dominios es una empresa encargada de registrar y gestionar dominios web.

  • Compra tu dominio web: Tras confirmar que el nombre de dominio preferido está disponible, se compra.

  • Verifica la titularidad del nombre de dominio: El último paso es verificar la información de contacto.

  • Proteger el dominio: Cuando se registra un dominio, la información de contacto pasa a estar disponible públicamente a través de la base de datos WHOIS para mayor transparencia. Como resultado, se es propenso a recibir mensajes de spam y ciberataques. Obtener protección de privacidad de dominio reduce el riesgo de fraude y robo de identidad al mantener oculta la información personal.

Diseño responsive

El diseño responsive es un enfoque de programación que facilita la adaptación automática de un sitio web al tamaño y la disposición de los dispositivos que utilizan los usuarios. Los sitios web responsivos pueden transformarse para proporcionar una experiencia óptima a los visitantes que acceden a las webs desde sus smartphones, tabletas y ordenadores de escritorio.

Este diseño adaptable reduce el tiempo de desarrollo, previene la duplicación de contenidos y potencia la viralidad de los mismos facilitando su difusión de manera más rápida y fluida.

Como se obtiene un diseño responsive

Se obtiene un diseño responsivo utilizando una combinación de cuadrículas flexibles, diseños fluidos y consultas de medios en CSS, que permiten a un sitio web adaptarse automáticamente al tamaño y la orientación de cualquier dispositivo. Esto se logra con porcentajes en lugar de píxeles fijos, escalado de imágenes, optimización de tipografías, y diseño pensando primero en el móvil (enfoque "móvil primero").

Los requerimientos de la solución web, los actores involucrados y la información a registrar

Requerimientos funcionales

Página de Información General: El sitio web debe mostrar información sobre la firma, la trayectoria, los servicios que ofrece, noticias legales relevantes, perfiles de los profesionales, reconocimientos y alianzas. También debe contar con la ubicación de las oficinas (Incluye mapa que indique las sedes) y una sección con los datos de contacto. Adicional debe contar con un formulario para que los clientes que deseen ser contactados dejen su información.

Gestión de Candidatos: La pagina debe contar con una sección llamada profesionales, el cual debe contener:

  • Un formulario para que los profesionales interesados en trabajar con la firma puedan registrar su hoja de vida, una vez realice el formulario este genere un código de radicado con el cual pueda solicitar información sobre la aplicación
  • Una sección donde solo inicien sesión los administradores y puedan revisar y evaluar las hojas de vida recibidas.
  • La capacidad de marcar si un candidato es aceptado o no, y esta a su vez contacte al candidato informando su negación o aceptación

Gestión de Casos Legales:
Dentro de la página debe contener una sección donde los profesionales puedan iniciar sesión, una vez inicien sesión la página debe contener:

  • Una sección para que los abogados registren información de sus clientes, incluyendo datos personales.
  • Una sección para registrar el tipo de proceso legal que se está manejando para cada cliente.
  • Una función para registrar y actualizar los avances de cada caso.


Requisitos no funcionales

  • Autenticación y Autorización: Solo los administradores y profesionales de la firma deben tener acceso a las secciones de gestión de candidatos y casos.

  • Protección de Datos: Se debe garantizar la confidencialidad de la información de los clientes, los candidatos y los procesos legales. Se deben implementar medidas de cifrado y protección contra accesos no autorizados.

  • Cumplimiento Legal: La solución debe adherirse a las leyes de protección de datos personales vigentes en Colombia.

  • Diseño: El sitio web debe tener un diseño profesional y acorde con la imagen de la firma.

  • Navegación: La navegación debe ser clara y lógica, permitiendo a los usuarios encontrar la información de manera rápida y sin confusión.

  • Velocidad de Carga: Las páginas del sitio web deben cargar rápidamente, incluso con una gran cantidad de contenido.

  • Tiempo de Respuesta: Las operaciones como el registro de datos o la búsqueda de información deben ejecutarse en un tiempo aceptable.

  • Tiempo de Actividad: El sitio web debe tener un alto porcentaje de tiempo de actividad, minimizando las interrupciones del servicio.

  • Gestión de Crecimiento: El sistema debe ser capaz de manejar un aumento en el número de usuarios, candidatos, clientes y casos.

  • Compatibilidad con Navegadores: La aplicación debe funcionar en diferentes navegadores
    • .
  • Diseño Adaptable: El sitio web debe visualizarse en computadores de escritorio, tabletas y teléfonos móviles.


Actores Involucrados

  • Público General: Visitantes del sitio web.

  • Candidato: Profesionales del derecho que desean postularse para trabajar con la firma.

  • Administrador / Profesional de la Firma: Son los abogados y el personal administrativo de "Te apoyamos SAS". Tendrán acceso a la gestión de hojas de vida, clientes y procesos legales.

  • Desarrollador web: El encargado de realizar la aplicación web

Información a registrar


• La compañía: Historia, misión, visión, servicios ofertados por la firma(información detallada)

• Noticias de interés de orden legal para el publico: Título, contenido, fecha de publicación.

• Perfil de los profesionales: Nombre, especialidad, experiencia, ubicación,foto, reconocimientos.

• Ubicación: Ubicación geográficas de cada una de las sedes en el pais

• Alianzas y Reconocimientos: Nombre de la entidad, tipo de alianza o reconocimiento.

• Contacto: Correo electrónico, número de teléfono, dirección, formulario de contacto.

• Información de Candidatos (Hoja de Vida): Datos Personales tales como nombre completo, número de identificación, correo electrónico, número de teléfono. Títulos, institución, año de graduación. Experiencia Profesional empresas en las que ha laborado, cargo, período, funciones. (Formulario)

• Estado de la Postulación: Aceptado/Rechazado.

• Información de Clientes y Casos: Datos del Cliente tales como nombre completo, número de identificación, correo electrónico, número de teléfono.

• Datos del Proceso: Tipo de proceso,fecha de inicio, abogado asignado, estado (activo/finalizado).

• Avances del Caso: Fecha del avance, descripción del avance, documentos adjuntos

Frameworks

Un framework representa una colección de diferentes clases que cooperan entre sí y que determina, así, la estructura básica de cada software que va a ser desarrollado tomando a este framework como base. Cuando un framework sirve como marco esencial para aplicaciones web, se puede hablar en este caso de frameworks para aplicaciones web, cuya forma abreviada es framework web. (IONOS, 2022)

Como base para el desarrollo de software, los web frameworks deben ofrecer estructuras sencillas y claras que faciliten su mantenimiento y que hagan posible que los desarrolladores puedan crear aplicaciones web en un corto período de tiempo.

Tres principios básicos de diseño:

• Don’t repeat yourself (DRY): el principio de no repetirse tiene que ver con evitar la redundancia. La información redundante como, por ejemplo, los códigos duplicados, que pueden evitarse, tienen un efecto negativo en el mantenimiento del software, ya que, en caso de tener que adaptar los fragmentos de código, los cambios deben llevarse a cabo en diferentes partes del código del programa.

• Keep it short and simple (KISS): a un problema, una solución. Este es el principio básico del paradigma KISS, que está orientado al principio de parsimonia o de economía: si se encuentra más de una solución para una determinada situación, se debería escoger aquella que requiera el menor número de hipótesis y variables. Para utilizar un framework o para solucionar un problema específico debería utilizarse la menor cantidad de código posible.

• Convention over configuration: las convenciones ofrecen la posibilidad de reducir la complejidad. Por ello, los frameworks web deberían fijar procedimientos acreditados en forma de configuración predeterminada y ofrecer, opcionalmente, otras posibilidades de configuración.

Aquí están algunos frameworks y herramientas que te pueden ayudar a crear una página web con usabilidad y diseño responsivo para la firma de abogados.


Frameworks de CSS

Estos frameworks facilitan la creación de diseños adaptables

• Bootstrap: Ofrece una cuadrícula robusta y componentes prediseñados (botones, formularios, menús de navegación) que se adaptan a diferentes tamaños de pantalla. Es ideal para un desarrollo rápido y eficiente.

• Foundation: Es conocido por ser flexible y semántico. Proporciona herramientas para crear diseños responsivos complejos.

• Tailwind CSS: Proporciona una colección de clases de utilidad que permiten construir diseños directamente en tu HTML.
